# 需求分析

本文档定义秦简牍动物字体识别基线的功能需求、数据要求和质量约束。

## 功能需求

| 编号 | 需求 | 当前状态 |
| --- | --- | --- |
| FR-1 | 从文件夹加载自定义动物字形数据集 | 已支持 |
| FR-2 | 生成并复用标签映射 | 已支持 |
| FR-3 | 训练分类模型 | 已支持 |
| FR-4 | 按验证集准确率保存最优权重 | 已支持 |
| FR-5 | 绘制训练损失和准确率曲线 | 已支持 |
| FR-6 | 单张图片推理 | 已支持 |
| FR-7 | 导出测试集指标与混淆矩阵 | 计划中 |
| FR-8 | 批量推理 | 计划中 |

## 数据要求

数据集必须采用以下目录结构：

```text
data/
├── train/<class_name>/*.png
├── val/<class_name>/*.png
└── test/<class_name>/*.png
```

支持的图片格式：

- `png`
- `jpg`
- `jpeg`
- `bmp`
- `webp`

最低质量要求：

- 单张图片应只包含一个主要字形或符号。
- `train`、`val`、`test` 中的类别文件夹应保持一致。
- 每个类别应有足够样本，避免模型只记住训练集。
- 图片裁剪应尽量贴近字形，同时保留完整笔画。
- 训练前应移除严重模糊、重复或标签错误的样本。

推荐起步规模：

| 阶段 | 每类样本量 |
| --- | --- |
| 冒烟测试 | 20+ |
| 初始基线 | 50+ |
| 有意义的对比实验 | 200+ |
| 较稳定实验 | 500+ |

## 模型要求

模型必须满足：

- 接收单通道图像。
- 输出形状为 `[batch_size, num_classes]` 的 logits。
- 类别数从训练数据集中自动推断。
- 训练和推理使用相同配置加载模型。

当前模型选择：

- `simple_cnn`：适合 `28 x 28` 输入的快速基线。
- `resnet18`：更强的单通道骨干，适合更大尺寸图像。

## 配置要求

核心配置位于 `src/config.py`：

```python
data_root = "data"
model_name = "simple_cnn"
image_size = 28
batch_size = 512
epochs = 100
lr = 1e-3
save_path = "checkpoints/best_model.pth"
```

每次实验至少记录：

- 数据集版本或来源
- 模型名称
- 输入尺寸
- batch size
- 学习率
- 训练轮数
- 最佳验证集准确率
- 权重保存路径

## 质量要求

| 方面 | 要求 |
| --- | --- |
| 可复现性 | 尽可能固定随机种子 |
| 可维护性 | 训练、模型、数据集、工具函数分离 |
| 可追踪性 | 每个数据集版本保留 `label_map.json` |
| 健壮性 | 缺少 `data/train` 时应清晰报错 |
| 可移植性 | 通过 PyTorch device 选择支持 CPU 和 CUDA |

## 非目标

当前基线不要求：

- 从完整简牍图片中检测文字区域
- 识别多字符文本行
- 构建 Web 服务
- 训练大规模基础模型
- 支持任意 OCR 输出格式
