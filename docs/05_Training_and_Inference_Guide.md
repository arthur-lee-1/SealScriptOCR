# 训练与推理指南

## 1. 文档目的

本文档用于说明本项目从环境准备、模型训练到单张图像推理的基本使用流程，便于后续实验复现与功能演示。

---

## 2. 环境准备

建议使用 Python 3.9 及以上版本。

推荐先创建虚拟环境，再安装项目依赖。

示例流程如下：

    python -m venv .venv

Windows 激活虚拟环境：

    .venv\Scripts\activate

Linux 或 macOS 激活虚拟环境：

    source .venv/bin/activate

安装依赖：

    pip install -r requirements.txt

---

## 3. 项目运行前检查

在开始训练前，请确认以下内容：

1. 已成功安装 Python 与依赖库
2. 项目目录结构完整
3. `src/` 下各模块文件已创建
4. 训练脚本与推理脚本能够正常导入模块
5. `checkpoints/` 与 `outputs/` 目录存在

---

## 4. 数据准备

当前项目以 MNIST 为实验数据集。
首次运行训练脚本时，程序通常会自动下载数据集。

默认数据处理包括：

- 图像转 Tensor
- 灰度图归一化
- 构建训练集与测试集 DataLoader

若后续替换为小篆单字数据集，则需要额外准备以下内容：

- 按类别划分的图像目录
- 类别名称与标签映射
- 统一尺寸的单字图像
- 训练集与测试集划分

---

## 5. 模型训练

### 5.1 使用训练脚本

训练命令示例：

    python src/train.py

如果项目支持命令行参数，也可以采用如下形式：

    python src/train.py --model simplecnn --epochs 10 --batch_size 64 --lr 0.001

### 5.2 训练过程说明

训练时一般会执行以下步骤：

1. 加载训练集和测试集
2. 初始化模型
3. 定义损失函数
4. 定义优化器
5. 进入 epoch 循环
6. 每轮输出训练损失与测试准确率
7. 保存最佳模型参数

### 5.3 训练输出内容

训练过程中，通常会输出如下信息：

- 当前 epoch
- 当前 batch 或平均 loss
- 当前测试集 accuracy
- 最优模型是否更新

示例日志格式如下：

    Epoch 1/10 | Train Loss: 0.2451 | Test Acc: 92.30%
    Epoch 2/10 | Train Loss: 0.1184 | Test Acc: 95.67%
    Epoch 3/10 | Train Loss: 0.0821 | Test Acc: 97.01%

---

## 6. 模型保存

训练完成后，模型权重建议保存在 `checkpoints/` 目录下。

常见保存方式如下：

- `latest_model.pth`：保存最新一轮模型
- `best_model.pth`：保存测试准确率最高的模型

建议同时保存以下信息：

- 模型参数
- epoch
- 优化器状态
- 当前最佳准确率

这样便于后续继续训练或进行推理。

## 7. 模型推理

## 7.1 单张图像推理

推理命令示例：

    python src/infer.py --img path/to/image.png --weights checkpoints/best_model.pth

程序执行流程一般包括：

1. 加载训练好的模型权重
2. 读取输入图像
3. 执行与训练一致的预处理
4. 进行前向推理
5. 输出预测类别和概率

### 7.2 推理结果示例

输出示例如下：

    Predicted class: 7
    Confidence: 0.9864

如果支持 top-k 输出，也可以显示为：

    Top-3 predictions:
    1. class 7 | 0.9864
    2. class 1 | 0.0081
    3. class 9 | 0.0032

## 9. 常见问题与解决方法

### 9.1 模块导入失败

问题表现：

    ModuleNotFoundError

可能原因：

- 当前运行路径不正确
- `src/` 目录未加入导入路径
- 文件结构与代码中的导入方式不一致

解决建议：

- 在项目根目录下运行命令
- 检查相对导入与绝对导入写法
- 确保文件名与模块名一致

### 9.2 GPU 不可用

问题表现：

程序未使用 CUDA，而是运行在 CPU 上。

解决建议：

- 检查 PyTorch 是否安装了 CUDA 版本
- 使用 `torch.cuda.is_available()` 检查 GPU 可用性
- 若无 GPU，可先使用 CPU 完成实验

### 9.3 loss 不下降

可能原因：

- 学习率设置不合理
- 模型结构过弱
- 数据预处理有误
- 标签错误
- batch size 过大或过小

解决建议：

- 调整学习率
- 检查输入图像和标签是否匹配
- 先在小批量数据上进行过拟合测试
- 简化问题逐步排查

### 9.4 推理结果异常

可能原因：

- 推理时预处理与训练时不一致
- 加载了错误的模型权重
- 输入图像尺寸不匹配
- 图像通道数不一致

解决建议：

- 保证训练与推理使用相同的 transform
- 检查模型类别数是否一致
- 检查输入图像格式

## 10. 从 MNIST 迁移到小篆识别的操作建议

当后续切换到小篆单字图像数据集时，建议按以下步骤修改：

1. 将数据加载方式改为 `ImageFolder` 或自定义 Dataset
2. 将类别数从 10 改为实际小篆类别数
3. 调整输入图像尺寸，例如 64×64 或 128×128
4. 根据图像特点增加增强策略
5. 保持训练与推理接口不变

建议新增的预处理包括：

- 灰度化
- 尺寸统一
- 二值化或对比度增强
- 随机旋转
- 随机平移
- 轻微缩放
