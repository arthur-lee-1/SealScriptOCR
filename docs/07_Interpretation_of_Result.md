# 结果解读

本文档说明如何解读秦简牍动物字体分类实验的训练输出。

## 输出产物

训练后检查：

```text
checkpoints/best_model.pth
outputs/training_plot.png
data/label_map.json
```

`training_plot.png` 包含损失和准确率曲线。`label_map.json` 用于确认类别顺序。

## 健康的训练现象

一个有效基线通常会表现为：

- 训练损失随 epoch 下降。
- 验证损失下降或趋于稳定。
- 训练准确率提升。
- 验证准确率提升后进入平台期。
- 明显测试图片可以得到正确标签。

小幅波动是正常的，尤其是在小数据集上。

## 常见曲线模式

| 现象 | 含义 | 建议 |
| --- | --- | --- |
| 训练 loss 几乎不变 | 模型没有学到东西 | 检查标签、学习率、输入张量 |
| 训练准确率高，验证准确率低 | 过拟合 | 增加数据、减少轮数、改进划分 |
| 训练和验证都低 | 欠拟合或数据质量差 | 检查样本和预处理 |
| 验证准确率大幅波动 | 验证集太小或噪声大 | 扩大验证集、修正标签 |
| 预测集中在一个类别 | 类别不平衡或标签问题 | 检查 `label_map.json` 和样本数 |

## 准确率

准确率是第一阶段基线指标：

```text
预测正确的样本数 / 总样本数
```

只有在以下条件一致时，准确率才适合用于比较实验：

- 数据划分不变
- 类别定义不变
- 预处理方式可比
- 使用同一个验证集

如果类别不平衡，整体准确率可能掩盖少数类表现。后续应增加每类准确率。

## 损失

损失用于判断训练行为：

- loss 下降说明优化器正在找到有效参数。
- 训练 loss 低但验证 loss 高，通常意味着过拟合。
- loss 噪声大，可能是学习率过高或 batch size 太小。
- loss 长期接近初始值，通常表示标签或预处理有问题。

## 标签映射

每次训练后都应检查 `data/label_map.json`：

```json
{
  "马": 0,
  "牛": 1,
  "犬": 2
}
```

如果这里的标签错误，即使模型正常训练，推理输出也会误导使用者。

## 错误分析

预测错误时，先判断错误来源：

| 原因 | 示例 |
| --- | --- |
| 字形歧义 | 动物字形之间共享相似笔画 |
| 裁剪不佳 | 字符被切断或主体太小 |
| 噪声干扰 | 简牍纹理盖过字形 |
| 标签错误 | 文件夹名与图片内容不一致 |
| 类别不平衡 | 稀有类别样本过少 |
| 风格偏移 | 训练集和验证集来自不同来源 |

每次实验记录 5-10 个典型失败样本。很多时候，这比直接换更深模型更有价值。

## 模型对比

对比 `SimpleCNN` 和 `ResNet18` 时，关注：

- 最佳验证准确率
- 收敛速度
- 过拟合差距
- 训练耗时
- 多次运行稳定性
- 失败样本质量

只有在验证表现提升且没有掩盖数据问题时，更强模型才算真正更好。

## 结果报告示例

```text
模型：resnet18
数据集：qin_animals_v1
最佳验证准确率：0.842
主要混淆：马 vs 鹿，牛 vs 羊
可能原因：上部笔画结构相似，裁剪边距不一致
下一步：清洗标签并增加每类混淆矩阵
```

## 后续改进顺序

1. 修正标签和重复样本。
2. 平衡类别数量。
3. 改进裁剪一致性。
4. 增加每类评估。
5. 加入保守的数据增强。
6. 对比更强模型骨干。
