# 开发流程

## 1. 开发策略

本项目采用渐进式开发策略：

先在 MNIST 上构建一个完整可运行的单字分类流程，
然后逐步改进模型，并为未来迁移到小篆单字识别做好准备。

开发过程强调以下原则：

- 尽早让代码跑起来
- 先构建完整流程，再逐步完善
- 逐步提升模块化程度
- 保持接口可复用

---

## 2. 开发阶段

## 第一阶段：项目初始化

### 目标

完成环境搭建并创建项目结构。

### 任务

1. 创建 Python 虚拟环境
2. 安装所需依赖
3. 创建项目目录
4. 初始化 `README.md` 和 `docs/`

### 输出结果

- 可运行的项目骨架
- 依赖安装完成
- 清晰的目录结构

---

## 第二阶段：数据集模块开发

### 目标

实现 MNIST 数据加载，并验证 batch 数据格式。

### 任务

1. 下载 MNIST
2. 编写 `mnist_dataset.py`
3. 定义图像变换
4. 创建训练集与测试集的 DataLoader
5. 检查样本 shape

### 预期结果

- 图像 batch shape：`[batch_size, 1, 28, 28]`
- 标签 batch shape：`[batch_size]`

### 输出结果

- 可用的数据集模块
- 必要时可视化部分样本

---

## 第三阶段：基础模型开发

### 目标

实现一个简单 CNN，并验证前向传播是否正确。

### 任务

1. 编写 `simple_cnn.py`
2. 定义卷积层、激活层、池化层和全连接层
3. 测试模型前向传播
4. 验证输出 shape

### 预期结果

- 输入 shape：`[B, 1, 28, 28]`
- 输出 shape：`[B, 10]`

### 输出结果

- 可运行的基线 CNN 模型

---

## 第四阶段：训练流程开发

### 目标

构建完整的训练与验证流程。

### 任务

1. 编写 `train_eval.py`
2. 编写 `train.py`
3. 定义损失函数
4. 定义优化器
5. 实现 epoch 循环
6. 输出训练日志
7. 保存最优模型

### 输出结果

- 完整训练脚本
- 每轮训练日志
- 保存的模型权重文件

---

## 第五阶段：推理模块开发

### 目标

支持使用训练好的模型进行单张图像预测。

### 任务

1. 编写 `infer.py`
2. 加载模型权重
3. 预处理输入图像
4. 执行前向推理
5. 输出预测类别和概率

### 输出结果

- 命令行推理脚本
- 可复用的单图预测流程

---

## 第六阶段：可视化与结果分析

### 目标

增加实验结果可视化与分析能力。

### 任务

1. 绘制训练损失曲线
2. 绘制验证准确率曲线
3. 可选生成混淆矩阵
4. 分析错误分类样本

### 输出结果

- 保存在 `outputs/` 中的图像结果
- 可解释的实验结果

---

## 第七阶段：升级为 ResNet18

### 目标

使用更强的模型替换 SimpleCNN。

### 任务

1. 编写 `resnet18_mnist.py`
2. 修改第一层卷积以适配单通道输入
3. 修改最后全连接层以输出 10 类
4. 复用相同训练框架
5. 与 SimpleCNN 进行效果对比

### 输出结果

- ResNet18 的训练结果
- 与 SimpleCNN 的基线对比结果

---

## 3. 建议开发顺序

为了降低调试难度，推荐按以下顺序实现：

1. `src/config.py`
2. `src/datasets/mnist_dataset.py`
3. `src/models/simple_cnn.py`
4. `src/utils/train_eval.py`
5. `src/train.py`
6. `src/infer.py`
7. `src/utils/visualize.py`
8. `src/models/resnet18_mnist.py`

## 5. 开发原则

在实现过程中，应遵循以下原则：

1. 第一版不要过度复杂化
2. 保证每个阶段都能运行
3. 每个模块都应独立测试
4. 记录错误与调试过程
5. 训练逻辑与推理逻辑要解耦

---

## 6. 阶段验收标准

每个阶段都应有明确的验收标准。

### 初始化阶段

- 环境能够成功运行
- 必要文件与目录已创建

### 数据阶段

- 能加载 MNIST
- 输入与标签 shape 正确

### 模型阶段

- 模型前向传播正常
- 输出维度正确

### 训练阶段

- loss 能随训练下降
- 能计算准确率
- 能保存最优模型

### 推理阶段

- 能成功加载模型
- 能对单张图像进行预测

### 可视化阶段

- 能生成 loss 和 accuracy 曲线

---

## 7. 开发流程最终目标

完成该开发流程后，项目应形成一个完整的单字识别基线系统，具备以下能力：

- 标准化的项目结构
- 完整训练流程
- 模型保存与加载
- 命令行推理
- 实验结果可视化
- 后续迁移到自定义小篆数据集的能力
